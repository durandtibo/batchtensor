# Security Policy

## Supported Versions

We release patches for security vulnerabilities. Currently supported versions:

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |
| 0.0.x   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability in `batchtensor`, please report it by emailing the maintainer at durand.tibo+gh@gmail.com.

**Please do not report security vulnerabilities through public GitHub issues.**

When reporting a vulnerability, please include:

- A description of the vulnerability
- Steps to reproduce the issue
- Potential impact of the vulnerability
- Suggested fix (if available)

We will acknowledge your email within 48 hours and provide a more detailed response within 5 business days indicating the next steps in handling your report.

After the initial reply to your report, we will keep you informed of the progress towards a fix and may ask for additional information or guidance.

## Security Update Process

When a security vulnerability is confirmed:

1. A fix will be developed and tested privately
2. A security advisory will be published
3. A new version will be released with the fix
4. The vulnerability details will be disclosed after users have had time to update

## Preferred Languages

We prefer all communications to be in English.
